- **OpenCode / Gemini CLI config**: smriti tools now advertise correctly via MCP. OpenCode and Gemini CLI need their own MCP server config entries pointing to `http://127.0.0.1:7333/mcp`. These live in their respective config files, not `~/.claude/settings.json`.

## Context

- The scan reads each file exactly once now (rayon parallel phase does hash + metadata + FTS together). Memory usage during scan is higher because DocInfo structs carry pre-computed metadata for all hashed files. For extreme cases, reduce `SMRITI_SCAN_BATCH_SIZE`.
- PRAGMAs are set aggressively during scans (`synchronous=NORMAL`, 64MB cache, 256MB mmap) and restored to defaults after. Error paths also restore.
- `flush_batch` uses `prepare_cached` — the 9 SQL statements are parsed once and reused across all entries in a batch.
