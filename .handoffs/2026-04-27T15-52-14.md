### Uncommitted changes

2. **`src/scanner.rs`** — Added `!entry.file_type().is_file()` check to skip non-regular files (sockets, pipes, devices) before attempting to hash them. Eliminates the noisy "cannot hash" warnings for files like Unix sockets.

### What was debugged

`smriti scan` crashed with SIGBUS (Bus error, core dumped). Backtrace: `walFindFrame -> readDbPage -> fts5NextMethod` in SQLite's WAL code. Root cause: stale committed WAL frames from a previous scan remain un-checkpointed; the next scan's large transaction (~200MB WAL, 50k+ frames) appends on top, triggering SIGBUS in SHM hash-table lookups.

FTS5 data is NOT corrupt (integrity check passes, rebuild works, read queries succeed). The issue is purely WAL lifecycle management.

### Future improvements (not blockers)

- **Transaction batching**: scan creates a single massive transaction (~200MB WAL for /home/josh). Batching into smaller commits would reduce WAL pressure.
- **rusqlite upgrade**: currently 0.31, latest is 0.39. Eight major versions behind; newer bundled SQLite may have WAL fixes.
- **roots remove bug** from previous handoff still open: allows removing non-existent roots without error.
