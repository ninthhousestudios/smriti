## blockers / context for next session

- None blocking. Working tree is clean except for `.handoffs/` archives
  and the `.sutra/` workspace dir.
- `smriti-serve` runs as a systemd user service. Restart with
  `systemctl --user restart smriti-serve` after any binary update so

  connection for its lifetime. If anyone adds a tool to mcp.rs that

  hold it on the persistent `Arc<Mutex<Connection>>`.
